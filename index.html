Baa.pl

use BLAT to ASSESS an ASSEMBLY

This program parses the output of a BLAT run of transcriptome vs. a genome. It returns 1) Total % coverage, 2) Average number of contigs mapped per mapped transcript, and 3) number of potential rearrangements. It will also create a file with transcript IDs that occur at regions that are potentially misassembled. It is simple to run. Just run the blat (transcripts vs. genome) and then run the script with the blat-output and your transcript-fasta-file as arguments.

Welcome to the baa.pl GitHub Page.

There are two main uses of this script.

  1. It is useful to use it to compare alternative assemblies. Maximizing how well transcripts align to an assembly is an independent criteria that can be used along with N50 and other stats to objectively choose between assemblies.
  2. It can be used to measure the completeness of an assembly in terms of coding sequence. This can be especially useful if you have an assembly that has lots of collapsed repeats so the assembly sizes are smaller than the estimated genome size.
A typical session with blat and baa.pl goes like this: 

$ blat genome_assembly1.fa ests.fa ests_v_genome_assembly1.blat
$ baa.pl ests_v_genome_assembly1.blat ests.fa > ests_v_genome_assembly1.baa
